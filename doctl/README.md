"# DigitalOcean Droplet Configuration\n\nThis folder contains all configuration files and documentation for deploying Guardr to DigitalOcean using doctl.\n\n## Files\n\n- `DigitalOcean-public.v2.yaml` - DigitalOcean API specification\n- `.do.env` - DigitalOcean API token\n- `README.md` - This file\n\n## Setup Instructions\n\n1. Authenticate with DigitalOcean:\n   ```bash\n   doctl auth init\n   # Use token from .do.env when prompted\n   ```\n\n2. Create Droplet:\n   ```bash\n   doctl compute droplet create guardr-prod \n     --image ubuntu-22-04-x64 \n     --region nyc3 \n     --size s-2vcpu-4gb \n     --ssh-keys <your-ssh-key-id> \n     --wait\n   ```\n\n3. Configure Firewall:\n   ```bash\n   doctl compute firewall create \n     --name guardr-firewall \n     --droplet-ids <droplet-id> \n     --inbound-rules \"protocol:tcp,ports:22,address:0.0.0.0/0\" \"protocol:tcp,ports:80,address:0.0.0.0/0\" \"protocol:tcp,ports:443,address:0.0.0.0/0\" \n     --outbound-rules \"protocol:tcp,ports:all,address:0.0.0.0/0\" \"protocol:udp,ports:all,address:0.0.0.0/0\"\n   ```\n\n4. Deploy Application:\n   - SSH into the droplet\n   - Clone the repository\n   - Run deployment scripts\n\n## Security Notes\n\n- API token is stored in .do.env (not tracked by git)\n- Use SSH keys for authentication\n- Enable automatic security updates\n- Configure firewall rules appropriately\n\n## Monitoring\n\n- DigitalOcean provides basic monitoring\n- Configure alerts for resource usage\n- Set up automated backups\n\n## Next Steps\n\n- Configure application deployment\n- Set up SSL certificates\n- Configure domain name\n- Set up monitoring and logging"
